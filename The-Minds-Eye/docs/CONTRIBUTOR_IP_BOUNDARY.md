# Contributor, IP, Privacy, and Source-Rights Boundary

This document is not legal advice. It is a project safety boundary for contributors and AI agents.

## Ownership Principle

The Mind's Eye should protect Adam Bell's ownership and project rights.

Do not add terms, licenses, outside datasets, contributor rules, or partner language that could weaken ownership without explicit human approval.

## Contributor Boundary

Before outside contributors, contractors, universities, museums, schools, or developers contribute meaningful work, the project should have a written agreement covering:

- who owns the contributed work;
- whether the contribution is assigned to the project owner;
- whether the contributor keeps any reuse rights;
- whether the work can be used commercially;
- whether the contributor can claim joint authorship;
- and whether the contributor can use the project name, data, or screenshots publicly.

## AI Agent Boundary

AI agents must not:

- add a repository license without approval;
- change ownership language;
- add open-source commitments;
- add model-training permissions;
- upload private student/school data;
- add third-party paid APIs without approval;
- ingest copyrighted content beyond allowed metadata or permitted excerpts;
- or imply that public historical source access means unrestricted reuse.

## Historical Source Rights

Every source record should eventually include:

- source title;
- source URL or archive ID;
- owning institution;
- rights statement if available;
- access date;
- allowed use notes;
- and whether full text, metadata only, or excerpt use is permitted.

When source rights are unclear, store metadata and citation references first. Do not store large copyrighted text blocks unless rights allow it.

## Student Privacy Boundary

The early prototype should avoid collecting student PII.

Do not build production student accounts, rosters, analytics, or saved student profiles until a privacy plan exists.

If classroom pilots begin, document:

- what data is collected;
- why it is needed;
- how long it is retained;
- who can access it;
- whether it includes student names, IDs, grades, or writing;
- and how deletion requests are handled.

## Teacher Authority Boundary

The teacher must remain the final instructional authority.

The system should support teacher review and approval before student-facing materials are used.

AI should guide, draft, organize, and cite. It should not silently replace teacher judgment.

## Grant Boundary

For non-dilutive funding, review all grant, university, accelerator, incubator, and partner terms for:

- ownership of IP;
- rights to derivative work;
- required open licensing;
- data sharing obligations;
- publication rights;
- commercialization restrictions;
- and first-right-of-refusal clauses.

If any term affects ownership or use rights, flag it for human/legal review before accepting.
