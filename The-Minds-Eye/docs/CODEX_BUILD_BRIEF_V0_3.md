# Codex Build Brief v0.3

## Immediate Goal

Prepare The Mind's Eye for a first working prototype that proves the learning model, not just the game model.

The first prototype should show that a teacher can:

1. select a TEKS
2. choose or create an authentic artifact requirement
3. attach a rubric
4. launch a source-grounded scenario
5. collect student evidence
6. see mastery status

## New First-Class Systems

### Assessment Artifact Framework

Build support for authentic products such as:

- newspaper articles
- editorials
- telegrams
- witness statements
- court statements
- lab reports
- engineering notebooks
- town council proposals

### Rubric System

Build support for:

- teacher-created rubrics
- uploaded/imported rubrics later
- rubric criteria and levels
- AI feedback recommendation
- teacher final score
- mastery tracking by TEKS

### Experience Engine

Build support for real-world labs and projects that feed the simulation.

Examples:

- water tests update the town water supply concern
- gardening experiments update harvest or market conditions
- engineering bridge tests update travel or commerce routes

## First Codex Sprint

Create a small CLI or minimal web prototype with these capabilities:

- define a TEKS placeholder
- create an artifact prompt
- attach a rubric
- submit sample student text
- generate AI feedback placeholder
- allow teacher score override
- record mastery attempt
- output a teacher-facing mastery summary

## Prototype Scenario

Example:

A town newspaper asks students to write an editorial using evidence from a historical source packet.

The teacher attaches a writing rubric.

The system records whether the student demonstrated the selected ELAR TEKS.

## Non-Negotiable

Do not build flashy gameplay before the assessment artifact and mastery evidence pipeline exists.

The platform must prove students are learning through authentic performance.
