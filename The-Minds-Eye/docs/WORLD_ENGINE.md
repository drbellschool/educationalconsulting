# World Engine

## Purpose

The World Engine turns historical records into a living instructional simulation.

It tracks the town, time, people, institutions, reputation, choices, consequences, inventory, and emerging quests.

## Core Concept

The Mind's Eye should not only generate a lesson. It should simulate a historically grounded town.

Quests emerge from:

- map locations
- train arrivals
- commerce needs
- church/community concerns
- court and sheriff activity
- newspaper reports
- weather and travel
- student choices
- teacher-selected standards

## World State

A world state should include:

- town
- year/date
- current time
- active map package
- active train schedule
- known institutions
- active NPCs
- active quests
- available inventory
- commerce conditions
- reputation states
- pending consequences
- TEKS objectives
- mastery requirements

## World Ledger

The world ledger records what happened in the simulation.

### Ledger Entries

- student/team action
- location visited
- item obtained or lost
- NPC interaction
- dice result
- source opened
- evidence submitted
- consequence triggered
- teacher override
- community note applied
- quest completed or failed

### Persistence Modes

- session_only: reset after one lesson
- unit_campaign: carries across lessons in a unit
- class_campaign: one class period has its own continuity
- archive_only: saved only for teacher review

## Quest Generation

Quests may be generated from:

- verified historical events
- map locations
- institutions
- train arrivals
- business shortages
- court notices
- church concerns
- student-generated problems
- teacher objective requests

Each quest must include:

- academic target
- source basis
- provenance classification
- map locations
- NPCs
- student actions
- possible consequences
- mastery evidence opportunities

## Reputation

The simulation may track reputation with:

- individual NPCs
- institutions
- town government
- merchants
- church/community groups
- sheriff/court
- railroad/depot staff

Reputation affects:

- access to information
- willingness to help
- cost of goods
- witness cooperation
- urgency of consequences

## Consequence Engine

The consequence engine responds to meaningful student choices.

It should never be used to shame students. It should create historically plausible civic learning moments.

Teacher approval should be required for major escalation.

## Standards Connection

Every generated or emergent branch should identify possible TEKS evidence moments.

Example:

A student team must persuade a judge using evidence.

Possible standards connections:

- social studies civic processes
- ELAR argument and evidence
- speaking/listening
- vocabulary acquisition

## Teacher Control

Teachers can:

- pause the world
- approve a branch
- reject a branch
- soften language
- trigger a checkpoint
- override AI scoring
- reset consequences
- preserve or discard ledger events

## Non-Negotiable

A living world is useful only if it remains source-aware, teacher-controlled, and instructionally purposeful.
