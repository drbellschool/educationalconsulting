# Mastery Through Action Model

## Purpose

The Mind's Eye must prove students are learning.

Students should not merely play through a simulation. The quests must require students to execute the targeted TEKS through authentic actions.

## Core Principle

A quest is valid only if the student must perform the standard to advance or succeed.

The simulation should avoid worksheet-like tasks, fake academic prompts, and disconnected quiz questions.

## TEKS Execution

For each selected TEKS, the system should define:

- required student behavior
- observable evidence
- acceptable response types
- success criteria
- number of successful demonstrations required
- support/scaffolding options
- teacher scoring controls

## Example: ELAR Evidence Standard

Instead of asking:

> What evidence supports the claim?

The simulation requires students to:

1. interview two witnesses
2. compare a newspaper clipping with a map or directory
3. identify which claim is better supported
4. present the argument to a judge, editor, town council, or community group
5. cite the evidence used

The dashboard records:

- evidence identified
- claim made
- source cited
- reasoning quality
- prompting needed
- mastery attempt count

## Example: Social Studies Civic Standard

Instead of asking students to define civic responsibility, they must:

- participate in a town meeting
- evaluate competing public needs
- use evidence from sources
- recommend a civic action
- explain consequences for different groups

## Example: Math Standard

Instead of a worksheet, students may:

- calculate train travel time
- compare freight costs
- estimate supply needs
- calculate change or budgets
- use map scale and distance
- evaluate probability in risk/travel checks

## Example: Science Standard

Students may:

- inspect water/fire infrastructure
- analyze disease or sanitation concerns
- evaluate materials and hazards
- reason about weather, flooding, or engineering
- design a solution using evidence

## Mastery Evidence Types

- written claim
- oral defense
- evidence notebook entry
- map annotation
- math calculation
- source comparison
- group decision with individual justification
- exit ticket
- teacher observation
- AI-captured student explanation
- artifact produced during simulation

## Mastery Attempt Model

Teachers set how many times a student must successfully demonstrate a standard.

Example:

```text
TEKS: selected standard
Required demonstrations: 3
Attempt 1: developing
Attempt 2: successful
Attempt 3: successful
Attempt 4: successful
Status: mastery met
```

## AI Scoring Role

AI can recommend, classify, and summarize evidence.

AI should not be the final authority.

Teacher can:

- accept AI recommendation
- override score
- add note
- require redo
- mark accommodation used
- attach evidence artifact

## Avoiding Fake Learning

The system should reject quests where:

- the TEKS is only mentioned but not required
- students can complete the mission without using the skill
- evidence is cosmetic
- the task is just a worksheet with a story wrapper
- dice results replace academic reasoning

## Quest Validation Test

Before a lesson launches, the system should ask:

1. What exact TEKS behavior must students perform?
2. Where does the student demonstrate it?
3. What evidence is captured?
4. How does the teacher know the student did it?
5. Could a student complete the quest without the standard?

If the answer to #5 is yes, the quest must be revised.

## Non-Negotiable

The game layer exists to create meaningful context for standards performance.

The standard cannot be decoration.
