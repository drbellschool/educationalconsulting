# Assessment Artifact Framework

## Purpose

The Mind's Eye must support authentic academic products that arise naturally from the simulation.

Students should write, calculate, investigate, argue, test, report, and present because the historical situation requires it, not because the platform hides a worksheet inside a game.

## Core Principle

The world creates a real reason to produce an academic artifact.

The teacher decides whether that artifact is practice, formative evidence, summative evidence, rubric-scored, AI-feedback-only, or teacher-scored.

## Artifact Types

Possible artifacts include:

- newspaper article
- editorial
- letter to the editor
- telegram
- public notice
- witness statement
- affidavit
- court statement
- closing argument
- judge recommendation
- town council proposal
- railroad incident report
- freight inspection report
- lab report
- field investigation report
- water quality report
- engineering notebook
- map annotation
- business proposal
- historical marker proposal
- oral presentation transcript
- debate notes
- scientific observation log
- exit ticket

## ELAR Writing Integration

ELAR teachers need students writing serious pieces that prepare them for state exams.

The simulation should create authentic audiences for writing.

Examples:

### Newspaper Request

A local newspaper asks students to submit:

- news article
- editorial
- eyewitness account
- investigative report
- letter to the editor

### Telegram or Letter

The railroad, sheriff, judge, mayor, editor, or business owner asks for:

- concise telegram
- formal letter
- incident explanation
- public statement

### Court or Law Scenario

The justice system asks students to write:

- witness statement
- affidavit
- claim with evidence
- defense statement
- closing argument
- recommendation to the judge

## Rubric Support

Teachers must be able to attach or build a rubric.

Rubric features:

- upload rubric
- create rubric manually
- select district/state writing rubric
- attach rubric to artifact type
- choose AI feedback only or AI score recommendation
- teacher final score/override
- track rubric dimensions over time

## Not All Writing Is Scored

Not every written piece should be graded.

The teacher can mark each artifact as:

- unscored practice
- AI feedback only
- formative evidence
- summative evidence
- rubric-scored
- teacher observation only
- mastery attempt

## Writing Feedback

Feedback should support:

- thesis/claim
- organization
- evidence
- elaboration
- coherence
- conventions
- audience/purpose
- use of historical source material
- TEKS alignment
- STAAR-style readiness where appropriate

## Artifact Record

Each artifact should include:

- artifact_id
- student_id or team_id
- session_id
- artifact_type
- prompt/context
- TEKS_ids
- rubric_id
- submission_text or file_url
- AI_feedback
- AI_score_recommendation
- teacher_score
- teacher_override_note
- mastery_status
- source_ids used
- provenance notes
- created_at

## Teacher Controls

Teachers can:

- choose which artifacts count for mastery
- set number of required successful demonstrations
- attach rubrics
- edit prompts
- require revision
- release AI feedback to students
- hide AI score from students
- override AI scoring
- compare class performance by rubric dimension

## Student Experience

Students should see why they are writing.

Bad:

> Write an essay about civic responsibility.

Better:

> The judge will not release the detained team unless someone submits a clear written statement explaining what happened, citing evidence, and proposing a fair remedy.

## Non-Negotiable

Writing must feel like it matters inside the world while still producing measurable standards evidence for the teacher.
