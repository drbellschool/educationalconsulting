# Tests

This folder will hold repeatability and source-integrity tests.

## Required Test Philosophy

The project should not merely produce creative missions. It should prove that missions are grounded in source evidence.

## Future Tests

1. A mission cannot label a claim as verified unless it has a source URL.
2. A mission must include a source integrity table.
3. Every playable location must have a stable location ID.
4. Fictional gameplay additions must be clearly labeled.
5. Source retrieval should save metadata before generation.
